# Heart Risk — offline Android ML app

Native Kotlin app (Jetpack Compose, Material 3, MVVM) that runs a **TensorFlow Lite** model trained on the **UCI Cleveland Heart Disease** dataset. All inference is **on-device**; there is no network usage and no WebView.

## Medical scope (important)

- The Cleveland dataset supports a **binary** target (presence of angiographic disease). It does **not** provide six independent disease-class labels.
- The app therefore reports:
  - **Risk tier**: Low / Medium / High (from calibrated probability thresholds).
  - **Likelihood** of the positive UCI label, with confidence.
- Six clinical categories (Normal, CAD, MI, Arrhythmia, Heart Failure, Cardiomyopathy) are shown as **educational context only** — they are **not** model outputs.

Always show the in-app disclaimer: *This is not a medical diagnosis. Consult a doctor.*

## Machine learning (`../ml`)

1. **Data**: UCI `processed.cleveland.data` (downloaded by the script).
2. **Preprocessing**: median imputation, `StandardScaler`, top-10 features by mutual information (on the training split only).
3. **Imbalance**: optional `--smote` on the training split.
4. **Models**:
   - **Random Forest** + `GridSearchCV` (baseline).
   - **XGBoost** + `GridSearchCV` (primary research model; metrics in `artifacts/metrics.json`).
   - **Keras** small MLP → exported to **`.tflite`** for mobile (same features as above).
5. **Artifacts** (after training):
   - `ml/artifacts/heart_risk_model.tflite`
   - `ml/artifacts/preprocessing_config.json`
   - `ml/artifacts/metrics.json`

### Train in Python (recommended: short-path venv on Windows)

```powershell
python -m venv C:\venv_tf
C:\venv_tf\Scripts\pip install -r ml/requirements.txt
C:\venv_tf\Scripts\python ml/train_and_export.py --smote
```

Copy outputs into the app:

- `heart_risk_model.tflite` → `HeartDiseaseApp/app/src/main/assets/`
- `preprocessing_config.json` → `HeartDiseaseApp/app/src/main/assets/`

If TensorFlow fails to install (long paths on Windows), enable **long paths** or use a venv under a short folder (e.g. `C:\venv_tf`).

## Android project

- **Package**: `com.heard.heartdisease`
- **Min SDK**: 26
- **Architecture**: MVVM, Room (history), Navigation Compose, CameraX (PPG heart-rate estimate only).

### Open in Android Studio

1. Open the `HeartDiseaseApp` folder.
2. Let Gradle sync; set `sdk.dir` in `local.properties` if needed.
3. Run on a device or emulator.

### Build APK / App Bundle

```text
Build → Build Bundle(s) / APK(s) → Build APK(s)
```

Or from a terminal (after Gradle wrapper is generated by Android Studio):

```text
gradlew.bat assembleRelease
```

Release builds use R8 minification; keep `proguard-rules.pro` if you add Gson models.

## Camera “scan” feature

The scan flow estimates **heart rate** from finger-on-camera PPG (brightness pulsatility). It does **not** infer disease. Users can apply the measured BPM to the **max heart rate** field before running the model.

## Optional extensions

- **Stacking ensemble** (RF + XGB + meta-learner): not included by default; add in `train_and_export.py` if you need it.
- **Optuna** tuning: swap `GridSearchCV` blocks for Optuna studies while keeping the same preprocessing export.
